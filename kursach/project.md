## ОСНОВНАЯ ЧАСТЬ

### 1. Анализ предметной области. Постановка задачи

#### 1.1. Описание предметной области и функции решаемых задач

**Предметная область:** Домашняя библиотека - персональная коллекция книг, принадлежащая семье или отдельному человеку. В отличие от публичных библиотек, домашняя библиотека имеет следующие особенности:
- Ограниченный круг пользователей (члены семьи, близкие друзья)
- Отсутствие коммерческой составляющей
- Необходимость контроля возрастного доступа
- Простые требования к учету

**Функции решаемых задач:**
1. **Учет книг:** Каталогизация всех книг в библиотеке
2. **Учет экземпляров:** Отслеживание физических копий книг
3. **Контроль выдачи:** Учет кто, когда и какую книгу взял
4. **Возрастные ограничения:** Автоматическая проверка соответствия возраста читателя и рейтинга книги
5. **Управление доступом:** Разграничение прав для разных пользователей
6. **Статистика:** Получение отчетов о состоянии библиотеки

#### 1.2. Перечень входных данных

1. **Данные о книгах:**
   - Название книги
   - Автор(ы)
   - Год издания
   - Издательство
   - ISBN
   - Жанр и категория
   - Возрастной рейтинг (0+, 6+, 12+, 16+, 18+)
   - Язык издания
   - Описание

2. **Данные о экземплярах:**
   - Инвентарный номер
   - Состояние (отличное, хорошее, удовлетворительное, плохое)
   - Место хранения
   - Дата приобретения
   - Стоимость

3. **Данные о пользователях:**
   - ФИО
   - Дата рождения
   - Контактная информация
   - Уровень доступа

4. **Данные о выдаче:**
   - Кто взял книгу
   - Какую книгу
   - Дата выдачи
   - Дата возврата
   - Состояние при возврате

#### 1.3. Перечень выходных данных

1. **Каталог книг:** Полный список всех книг с фильтрацией
2. **Список доступных книг:** Книги, которые можно взять
3. **Статистика библиотеки:**
   - Общее количество книг и экземпляров
   - Распределение по жанрам
   - Статистика выдачи
4. **Отчеты по пользователям:**
   - История выданных книг
   - Текущие выдачи
5. **Предупреждения:**
   - О нарушении возрастных ограничений
   - О просроченных возвратах

#### 1.4. Ограничения предметной области

1. **Законодательные ограничения:**
   - Федеральный закон № 436-ФЗ "О защите детей от информации"
   - Обязательная возрастная маркировка
   - Запрет выдачи книг 18+ лицам младше 18 лет

2. **Технические ограничения:**
   - Максимальное количество пользователей: 50
   - Максимальное количество книг: 10 000
   - Одновременных выдач: не более 5 на пользователя

3. **Бизнес-ограничения:**
   - Только для некоммерческого использования
   - Отсутствие системы штрафов
   - Упрощенный учет (без сложных финансовых операций)

#### 1.5. Взаимодействие с другими программами

1. **Веб-браузер:** Для доступа через веб-интерфейс
2. **Резервное копирование:** Интеграция с системами бэкапа
3. **Экспорт данных:** Возможность экспорта в CSV/Excel
4. **API:** Для интеграции с мобильными приложениями

### 2. Инфологическая (концептуальная) модель базы данных

#### 2.1. Выделение информационных объектов

В результате анализа предметной области были выделены следующие сущности:

1. **Книга (Book)** - описание книги как литературного произведения
2. **Автор (Author)** - создатель книги
3. **Издательство (Publisher)** - организация, выпустившая книгу
4. **Жанр (Genre)** - литературный жанр книги
5. **Категория (Category)** - крупный раздел литературы
6. **Экземпляр (Copy)** - физическая копия книги
7. **Пользователь (User)** - читатель библиотеки
8. **Выдача (Loan)** - факт выдачи книги пользователю
9. **Место хранения (Location)** - физическое расположение книги
10. **Язык (Language)** - язык издания
11. **Серия (Series)** - серия книг
12. **Возрастная категория (AgeCategory)** - возрастной рейтинг

#### 2.2. Определение атрибутов объектов

**Основные атрибуты сущностей:**

1. **Книга:**
   - ID (первичный ключ)
   - Название
   - Год издания
   - Количество страниц
   - ISBN
   - Описание
   - Возрастной рейтинг

2. **Автор:**
   - ID
   - Фамилия
   - Имя
   - Отчество
   - Год рождения
   - Страна
   - Тип (отечественный/зарубежный)
   - Статус иностранного агента

3. **Пользователь:**
   - ID
   - ФИО
   - Дата рождения
   - Email
   - Телефон
   - Уровень доступа (админ/библиотекарь/гость)

#### 2.3. Определение отношений и мощности отношений между объектами

1. **Книга - Автор:** Многие-ко-многим (одна книга может иметь нескольких авторов, один автор может написать несколько книг)
2. **Книга - Жанр:** Один-ко-многим (одна книга принадлежит одному жанру, в одном жанре может быть много книг)
3. **Книга - Экземпляр:** Один-ко-многим (одна книга может иметь несколько экземпляров, каждый экземпляр относится к одной книге)
4. **Пользователь - Выдача:** Один-ко-многим (один пользователь может иметь несколько выдач, каждая выдача относится к одному пользователю)
5. **Экземпляр - Выдача:** Один-ко-многим (один экземпляр может быть выдан несколько раз, каждая выдача относится к одному экземпляру)
